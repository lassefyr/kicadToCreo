Target of this project is to produce Creo Schematic compatible output to Creo parametic 3d program (Creo 4.0 M070).

PTC Creo is one of the "big" 3d parametric design softwares. Other similar products are Solidworks, Catia, Autocad Inventor, Onshape etc. One advantage of these programs is that they let you model every detail of your product. You can import pcb's using the IDF format and you can also route cables to your end product. The cad program will give you the needed cable lengths and cable drawings.

Creo has a built in Schematic diagramming software (Creo Schematics Lite) but, for us,  it is little bit too professional for simple cabling and wiring (+- 50 cables/product). Kicad, on the other hand, is quite simple to use and component libraries can be generated quickly and easily. Therefore I decided write a small script for Kicad that generates the needed logical connections file for Creo. Now we can use Kicad for all electrical design. Creo can be used for final 3d modeling.

Included with this repository are some Kicad schematic examples with the component library and some images as how simple designs look in Creo.

3. Back Annotate

After routing your cables you can back annotate wire lengths and Creo harness part names to wires and cables.

First in Creo you must export the logical connections (Creo Schematic xml) to a xml file. Name the file as your Kicad design filename +_creoin.xml. For example, if your kicad design is named mycables.sch then you need to name the creo logical file as mycables_creoin.xml.

From Kicad, load the script xmlReadCreo.py and run it. After running the script you will get a file named mycables_creo.sch. This design shoud contain the harness lengths and harness part nubers (or names).
